# 00 – プロジェクト概要とドキュメント構成

## 1. プロジェクト概要

- リポジトリ: `sentoku870/tumego`
- 種別: ブラウザで動作する **囲碁の詰碁／局面ビューア・エディタ**
- 主な機能（ver2〜ver3 想定）:
  - SGF 読み込み・盤面表示
  - 編集モードでの盤面編集（石配置・削除・初期図／問題図の切り替え など）
  - 解答モードでの着手入力・手数表示・簡易な履歴操作
  - 端末に応じた UI レイアウト（PC / スマホ縦 など）

このリポジトリは **Web 版のみ** を対象とし、PC ネイティブアプリや外部エンジンとの本格連携は「将来構想」として別扱いとする。

## 2. 対象範囲と非対象（non-goals）

このドキュメントセットは、次の範囲を対象とする。

- 対象:
  - Web 版アプリ本体（HTML / CSS / TypeScript）
  - GitHub 上のブランチ運用・リリースフロー
  - 軽量な分析機能やヒントモード（将来の ver3 以降）

- 当面の非対象（non-goals）:
  - 重い計算リソースを使う本格的な対局 AI エンジン連携（KataGo など）を Web 版に直接組み込むこと
  - フル機能の SGF エディタ（高度な変化図ツリー編集など）
  - PC 版（ローカルアプリ）固有の要件・UI

これらの non-goals は別途 `non-goals-and-scope` 系の資料で詳細に説明されるが、
**Web 版の ver2〜ver3 では上記を「やらないこと」として前提にする。**

## 3. アーキテクチャ概要（簡易）

実装はおおまかに次のレイヤに分かれる。

1. **UI 層**
   - `index.html`, `layout.css` など
   - 画面レイアウト、ボタン配置、レスポンシブ対応（PC / スマホ）を担当

2. **アプリケーションロジック / 状態管理**
   - TypeScript で実装されたモジュール群（例: GameStore, History 管理, Renderer など）
   - 編集モード / 解答モード、盤面状態、履歴、各種設定の状態を保持・更新する

3. **外部との I/O**
   - SGF 読み書きロジック
   - 将来的な「ヒントモード＋AnalysisEngine v1」などの軽量エンジン連携

詳細な構造や依存関係は、アーキテクチャ関連の資料（`アーキテクチャ概要.md`, `ver2 分析レポート.md` など）に記載されており、
必要に応じてそちらを参照する。

## 4. ドキュメント構成

`docs/` フォルダの主な役割は次の通り:

- `00-overview.md`（このファイル）
  - プロジェクト概要と、他のドキュメントの位置づけを説明する

- `10-branch-strategy.md`
  - main / stable-20251116 / feature ブランチ / codex ブランチなどの運用ポリシー

- `11-release-flow.md`
  - GitHub Pages 用の **安定版ブランチ (stable-20251116)** をどう更新するか
  - dist/ のコミットを含むリリース手順

- `12-dist-handling.md`
  - dist/ ディレクトリをいつ・誰が・どう扱うかのルール

- `20-codex-rules.md`
  - Codex / DeepResearch に任せてよい作業範囲
  - 禁止事項（stable や Pages 設定、dist 更新など）

その他の詳細設計（UI 設計ガイド、修正レベル詳細、Codex 依頼ガイド、PR 問題対応ガイド、将来の GoEngine 見直し・ヒントモード仕様など）は、
リポジトリ直下の md/txt ファイルとして配置し、必要に応じて `20-codex-rules.md` から参照する。
