## 1. 棋力レベル定義（G0-G4）

ユーザー本人の棋力は **野狐4-5段（G4相当）** を想定。
ただし、このアプリの対象ユーザーは **級位者〜初段（G0-G2）** が中心。

| ラベル | 目安棋力 | 特徴 |
|:------:|----------|------|
| G0 | 〜10級 | ルール〜基本死活。対局経験少なめ |
| G1 | 5〜1級 | 一般アマ級位者。布石・定石は一部のみ |
| G2 | 初段〜二段 | 基本形理解あり。読み抜け・手筋抜けあり |
| G3 | 三〜四段 | 戦い強いがムラあり。ヨセに取りこぼし |
| G4 | 五段相当 | ユーザー本人。1-2分野に明確な弱点 |

### 説明の使い分け
- G0-G1: 「何を気にすべきか」を絞った説明
- G2: 「なぜそうなるか」の簡潔な理由を添える
- G3-G4: 手順や構想の裏付けも欲しいが、文章量は抑える

## 2. 解説レベル定義（A-D）

棋力（G0-G4）とは独立に、「どこまで言語化するか」を定義。

| ラベル | 概要 | 具体例 |
|:------:|------|--------|
| A | 方向性ヒント | 「まず隅の石を助けましょう」 |
| B | 役割・構図ヒント | 「隅の黒2子を捨てると、外側の厚みが活きます」 |
| C | プロ構想レベル | 複数手先の構想と変化を文章で説明（重い） |
| D | AI解析レベル | あらゆる変化を網羅（非現実的、基本不採用） |

### このアプリでの対象
- **提供対象**: A / B のみ
- **対象外**: C / D（プロ・高段者向けの詳細解析）

## 3. 学習・解説コンセプト

### 3.1 2つの能力
囲碁の上達には2つの能力がある：

| 能力 | 説明 | このアプリでの扱い |
|------|------|-------------------|
| パターン認識 | 形を見て「こういう形はこう」と判断 | **主対象** |
| 読み | 手順を頭の中で追う | 補助的 |

### 3.2 対象ユーザー
- **主対象**: 級位者〜初段（G0-G2）
- **目的**: パターン認識の強化
- **スタイル**: NHK解説風のワンポイントコメント

### 3.3 解説スタイルの例

**良い例（レベルA）:**
```
「黒は左上の石を助けるのが急所です」
```

**良い例（レベルB）:**
```
「左上の黒2子を助けると、白の模様を削りながら安定できます」
```

**避けるスタイル:**
```
「ここで黒がA17に打つと、白はB18で応じ、黒C19、白D18、
黒E17となり、黒は2目半のリードを維持できます」
→ 数値や長い変化図は補助的な情報とする
```

## 4. 詰碁の基本用語

### 4.1 基本用語

| 用語 | 説明 |
|------|------|
| 詰碁 | 部分的な死活問題 |
| 問題図 | 解答前の初期局面 |
| 正解 | 最善の手順 |
| 失敗 | 目的を達成できない手順 |
| コウ | 取り返しの連続を防ぐルール |

### 4.2 死活用語

| 用語 | 説明 |
|------|------|
| 生き | 2眼を作って活きること |
| 死に | 2眼を作れず取られる状態 |
| セキ | 双方取れない状態 |
| 眼 | 相手が打てない空間 |
| 急所 | 最も効率の良い点 |

## 5. このアプリでの詰碁機能

### 5.1 編集モード（edit）
- 石の連続配置（黒白切替がスムーズ）
- 囲碁ルール ON/OFF
- 最終的には囲碁ルールに従った局面を作る

### 5.2 解答モード（solve）
- 問題図の固定
- 解答シーケンスの記録
- 番号付き画像への出力
- URL / QR コードでの共有

### 5.3 閲覧モード（view）
- SGF の再生
- 特定局面の問題図化
