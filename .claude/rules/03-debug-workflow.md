# デバッグワークフロー（Debug Workflow）- Web版

> このファイルはバグ報告・デバッグ作業の進め方を定義します。
> 再現→原因特定→修正→確認のサイクルを標準化します。

---

## 1. 役割分担

### ユーザー
- 症状の報告（いつ/どこで/何が起きたか）
- ブラウザ DevTools でコンソールエラーを確認・コピペ
- 指定された手順での再現操作
- スクリーンショット（表示崩れの場合）

### Claude Code
- 症状の整理と仮説立て
- 再現手順の設計
- 必要な情報の指示
- 修正レベル（Lv0-5）の判定
- 修正の実行（Plan Mode 経由）
- 確認手順の提示

---

## 2. デバッグサイクル（7ステップ）

### Step 1: 現象の整理
- **いつ**: どの操作の後か
- **どの画面/モード**: edit / solve / view のどれか
- **何が起きた**: エラー？フリーズ？表示崩れ？
- **期待していた動作**: 本来どうなるはずか

### Step 2: 再現手順の確定
```
1. ブラウザで https://sentoku870.github.io/tumego/ を開く
2. ○○モードに切り替える
3. 石を○○に置く
4. ○○ボタンを押す
5. → エラー発生
```

### Step 3: 情報収集
必要に応じてユーザーに依頼：
- **コンソールエラー**: DevTools → Console のエラー全文
- **ネットワークエラー**: DevTools → Network（必要な場合）
- **スクリーンショット**: 表示崩れの場合
- **デバイス情報**: PC / スマホ（iOS/Android）/ 画面向き

### Step 4: 原因候補の整理
| 候補 | 可能性 | 確認方法 |
|------|--------|----------|
| 状態管理のバグ | 高 | GameStore の状態をログ出力 |
| UIイベントの問題 | 中 | イベントハンドラにログ追加 |
| レスポンシブ崩れ | 中 | 別デバイスで再現するか |

### Step 5: 修正方針とレベル決定
```
修正レベル: Lv2（中程度）
Git フロー: feature/YYYY-MM-DD-fix-xxx → PR
```

### Step 6: 修正と確認
1. Claude Code が修正を実行
2. `npm run dev` でローカル確認
3. ユーザーが再現手順で確認
4. PC / スマホ両方で確認（必要に応じて）

### Step 7: 引継ぎメモ作成（任意）

---

## 3. Web版固有のデバッグポイント

### 3.1 よくある問題

| 症状 | 原因候補 | 確認方法 |
|------|----------|----------|
| 画面が真っ白 | JS エラー | DevTools Console |
| ボタンが反応しない | イベントハンドラ | Console ログ |
| 石が置けない | GameStore 状態 | 状態ログ出力 |
| スマホで表示崩れ | CSS / レスポンシブ | 別デバイスで確認 |
| localStorage 関連 | 保存/読込エラー | Application タブ |

### 3.2 DevTools の使い方

**コンソールエラーの取得:**
1. F12 または Ctrl+Shift+I で DevTools を開く
2. Console タブを選択
3. 赤いエラーメッセージをコピー

**ネットワーク確認:**
1. Network タブを選択
2. 問題の操作を再現
3. 赤いリクエストがあれば詳細を確認

**モバイルエミュレーション:**
1. DevTools の左上のデバイスアイコンをクリック
2. デバイスを選択（iPhone, Galaxy など）
3. 画面サイズ・向きを変えて確認

### 3.3 関連ファイル

| 症状の領域 | 確認するファイル |
|------------|------------------|
| 状態管理 | `src/core/GameStore.ts` |
| 履歴/Undo | `src/core/History.ts` |
| 盤面表示 | `src/ui/BoardView.ts` |
| ツールバー | `src/ui/Toolbar.ts` |
| 設定 | `src/ui/Preferences.ts` |
| SGF処理 | `src/core/SGFParser.ts` |

## 4. デバイス別の確認ポイント

### 4.1 PC（Chrome）
- マウス操作（クリック、ドラッグ）
- キーボード入力（テキストフィールド）
- ウィンドウリサイズ

### 4.2 スマホ縦（iOS / Android）
- タップ操作
- 盤面のサイズ・位置
- ボタンの押しやすさ
- ソフトキーボード表示時のレイアウト

### 4.3 スマホ横
- 盤面とボタンの配置
- 画面回転時の挙動

## 6. ver3 デバッグ一時停止中の注意

現在 ver3 デバッグは一時停止中（PC版優先）。

再開時の方針：
- まず現状の問題点を整理（T1/T2/T3 分類）
- T1（バグの温床）を最優先
- T2（将来機能の足かせ）を次に
- T3（気持ち悪さ）は余裕があれば
